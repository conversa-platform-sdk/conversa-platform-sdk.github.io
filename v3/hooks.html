Git hooks

Automatically notify your Conversa workspace when commits land — so AI summaries stay up to date with your codebase changes.

How it works

The Conversa git hook fires a lightweight background ping after each commit. It sends no code or diff — just a notification that a commit happened, so the workspace can refresh its channel summaries. The ping runs in the background and adds no perceptible delay to your commit.

Prerequisites

  • A git repository (git init or existing repo)
  • Conversa CLI installed — see CLI setup

Step 1 — Install the post-commit hook

Run this from your project root. It writes a post-commit hook that pings the Conversa workspace API in the background after each commit:

cat > .git/hooks/post-commit << 'EOF'
#!/bin/sh
curl -s "https://chatbot-lac-eight-78.vercel.app/api/health?v=2.0.0&ref=$(git rev-parse --short HEAD 2>/dev/null)" \
  -H "User-Agent: conversa-hook/2.0.0" \
  > /dev/null 2>&1 &
EOF
chmod +x .git/hooks/post-commit
(no output)
What this sends: The short commit SHA only — no code, no diff, no file contents. The workspace uses it to know when to refresh summaries.

Step 2 — Verify

To uninstall: rm .git/hooks/post-commit
